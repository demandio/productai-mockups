01 / Context

Three Eras of Shopping

Shopping is undergoing a structural transformation. The surfaces we build today must survive a transition from human-driven browsing to agent-mediated commerce.

Now
~2027
~2029+
Era 1 / Now

Human-Driven, Tool-Augmented

Users browse, search, click. Extensions and apps augment. Product.ai surfaces are standalone tools the user opens deliberately.

Era 2 / ~2027-2029

Agent-Advised, Human-Confirmed

OS agents mediate shopping. Users set goals, agents do work, users confirm. Product.ai integrates via App Intents and MCP.

Era 3 / ~2029+

Agent-Autonomous, Human-Governed

Agents execute end-to-end. My Radar becomes the primary interface. Biometric Presence Gate blocks full autonomy on iOS.

What Survives Every Era

The intelligence layer - Truth Graph + Commerce Graph + Preference Graph. Surface-agnostic. Whether served by an app, extension, MCP server, or OS agent.
Verification infrastructure - Protocol commerce makes this MORE valuable. Structured rejection codes create richer signal than DOM observation.
The Preference Graph - Deepens over time. Creates genuine switching cost. OS agents start from zero with every user.

What Doesn't Survive

DOM injection - Permission ratchet, iframe isolation, and protocol commerce collectively end this model.
Installation as distribution - MCP servers, APIs, OS agent integrations reach users without installation.
Brute-force code testing - Protocol commerce replacement semantics and the Honey precedent make this extinct.

04 / Desktop Extension

Side Panel Architecture

A Chrome side panel that serves as your second opinion at the point of shopping. 380px alongside the merchant page. Reads the URL to identify the product, then serves everything from Product.ai's backend.

Invocation Model, Not Injection

The extension never touches the merchant page DOM. No content script injection, no overlays, no host permissions. Four structural pressures are killing the injection model:

Pressure 1

Permission Ratchet

Safari defaults to per-site "Ask" with 24-hour expiration. Chrome making per-site controls more prominent.

Pressure 2

Iframe Isolation

Payment processors moving card inputs into cross-origin iframes, shrinking accessible checkout surface.

Pressure 3

Declarative Inversion

MV3 replacing programmatic decisions with declarative rules.

Pressure 4

Protocol Commerce

UCP/ACP replacement semantics make brute-force code testing structurally extinct.

  5. App Intents / Siri integration - technical scoping needed. Priority for Era 2 readiness.
  6. WebMCP preparation - monitor Chrome 146+ for flag availability. Structure extension intelligence as typed tool interfaces now.
